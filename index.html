Course Synopsis

19 Dec, 2013 | Category: General | Author: Scott

Open source community based software is gaining traction in many fields. Packages such as Scientific Python, aided by collaborative coding paradigms like GitHub, allow multi-institution multi-agency groups to create naturally evolving software projects. The course has two main objectives: It will introduce the participants to community software packages designed for radar data processing, and it will endow participants with the basics of how to contribute to community software.

Course structure and methodology

The course will be interactive and “hands-on” like a tutorial. Participants will be introduced to different version control systems (e.g. git, mercurial) and to online code sharing platforms such as GitHub (http://github.com) or Bitbucket (http://bitbucket.org). They will gain skills such as “forking” a repository, making, checking in and committing changes and submitting “pull requests”. This will enable participants to not only use community software but to actively contribute to community code.
The following sessions will be task-based, and will primarily use two Python-based radar toolkits: wradlib (http://wradlib.bitbucket.org/) and Py-ART (http://arm-doe.github.io/pyart/) with several other packages including BALTRAD. Covered aspects might include:
● reading polarimetric radial data and plotting and interrogating a single ray,
● plotting constant elevation (PPI) data,
● polarimetric phase processing including rainfall estimation,
● mapping and compositing to a cartesian grid,
● gauge adjustment,
● volume macrophysical statistics.
For the exercises, participants will use the iPython Notebook (http://ipython.org). The course organisers will make sure in time that the participants will receive detailed guidance on how to prepare their notebooks for the exercises.

Target audience and prerequisite knowledge

This course is aimed at scientists who are interested in exploring new tools for interacting with scanning radar data. Limited knowledge or familiarity with Python is useful, but not required.

Expected outcomes

After completing the course, attendees can expect to have a set of open source, community software recipes to complete everyday radar processing tasks. This course will not turn attendees into Python gurus but will help those who wish to take the first step into the world of open source community software. The course will be very “problem based” and participants will be encouraged to bring with them examples from their research which the workshop leaders will endeavor to use during the course.
